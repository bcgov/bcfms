### 1. BEFORE UPGRADING (!!! IMPORTANT !!!)
1. Update the maintenance page with appropriate dates and details. Maintenance page is found at `/sw_ux/httpd0(1|2)/www/html/maintenance.html`
2. Put system into maintenance mode:
```shell
touch /apps_ux/apps_(dlvr|test|prod)/maintenance/bcfms
```
3. Confirm system in maintenance mode by accessing:
   `https://(|dlvr|test)apps.nrs.gov.bc.ca/bc-fossil-management`. System should not be live, and maintenance page should be displayed.

4. Backup database:
```shell
sudo -u wwwadm /bin/bash
cd /apps_ux/apps_[env]/backups/database
chmod 700 .
pg_dump -Upostgres bcfms_[env] > 2026-03-04_bcfms_pre_1.3.0.sql
gzip 2026-03-04_bcfms_pre_1.3.0.sql
chmod 400 2026-03-04_bcfms_pre_1.3.0.sql.gz
chmod 500 .
```

5. Backup venv
   mv bcfms_<env>_venv backups/bcfms_<env>_venv.2026-03-03

### 2. Rollback patched migration
1. psql -Upostgres bcfms_<env> -c "delete from django_migrations where name = '12008_alter_file_path'"

### 3. Perform deployment
1. The source code branches are as follows (`cd/docker/config/app_env`):
@todo - Need to update core repo to create version
```yaml
config_branch: "v1.3.0"
core_repo_tag: "stable/8.0.8_bcgov_12377_12600_12616_12502" 
common_repo_tag: "v2.0.0"
```

2. The `migrations` variable must be set as follows (empty line ensure migrations are complete):
```yaml
perform_migration: False
```

3. Run the migration from the bcfms repo root:
```shell
./cd/docker/run_playbook.sh (dlvr|test|prod)
```

### 4. Deploy new version of application
```shell
sudo -uwwwadm /bin/bash
cd /apps_ux/apps_<env>
source bcfms_<env>_venv/bin/activate
cd bcfms
python manage.py migrate models
python manage.py migrate
python manage.py graph publish --update -ui
python manage.py check --deploy --settings=bcfms.settings
python manage.py validate
python manage.py validate --fix 1012
python manage.py bc_reindex_database
exit
```

### 4. Take system out of maintenance mode:
```shell
sudo -uwwwadm rm /apps_ux/apps_(dlvr|test|prod)/maintenance/bcfms
```
